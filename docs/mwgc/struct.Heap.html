pub struct Heap<'heap> { /* fields omitted */ }

Takes ownership of a block of Memory, hands out chunks of it, and garbage collects unused chunks on demand.

The heap is organized into allocatable blocks, with a small region reserved as a bitmap. The block size is configured by a compile-time constant, using a default of 16 bytes. Each block has a 2-bit "color" in the bitmap, at a cost of about 2% overhead. One "color" is used to mark continuations of contiguous spans, so each allocation can be identified by a color followed by one or more "continue" markers. The other 3 colors correspond to the white, gray, and black colors of tri-color marking, although I've chosen to call them blue, green, and "check".

Separately, a sorted free list is maintained by storing a "next" link and size in each free block. This consumes 8 bytes on a 32-bit system, limiting the minimum block size.

The heap object (its state) should consume about 9 words, or 36 bytes on a 32-bit system.

pub fn mark_start<T>(&mut self, roots: &[&T])
[src]

Start the first phase of garbage collection. This is only useful if you want tight control over latency -- otherwise, you should call gc().

roots must be a slice of references to objects in the heap which are the roots of your object graph. Successive calls to mark_round() will follow the object graph and mark all live objects. A follow-on call to sweep() will free the remaining spans.

Important: If you modify any objects before mark_round() returns true, you must notify the garbage collector to (re)check the object you modified by calling mark_check.

pub fn mark_round(&mut self) -> bool
[src]

Do one "round" of the mark phase of garbage collection. This is only useful if you want tight control over latency -- otherwise, you should call gc().

Walk the current mark span, following links from marked objects to find objects to check on the next round.

Returns true if this phase is over, and all live objects have been marked. You must call sweep() to finish the collection and free up memory.

Important: If you modify any objects before this function returns true, you must notify the garbage collector to (re)check the object you modified by calling mark_check.

pub fn mark<T>(&mut self, roots: &[&T])
[src]

Do the mark phase of garbage collection (the first of two phases).

roots must be a slice of references to objects in the heap which are the roots of your object graph. Any object that isn't directly or indirectly referenced by these roots will be freed by a following call to sweep().

This is the equivalent of heap.mark_start(roots); while !heap.mark_round() {};.
